**Local preview**: `bash deploy.sh && python -m http.server 8000`

## How It Works

IF Hub has two ways to play games:

### 1. Main Hub (app.html)

`app.html` is the full-featured interface:
- **Game selector dropdown** тАФ switches between all registered games
- **Source viewer tab** тАФ displays the game's `story.ni` with syntax highlighting
- **Walkthrough viewer tab** тАФ shows walkthrough guides (if available)
- **Centralized sound controls** тАФ mute button + volume slider (visible only for sound-enabled games)
- **Game player** тАФ an iframe that loads `play.html` with the selected game's binary

The game player is an iframe loading `play.html?binary=X&title=Y`. Sound-enabled games use native Glk/blorb sound тАФ audio is embedded in `.gblorb` binaries and played by the Parchment interpreter directly (no external JS needed).

### 2. Standalone Play Pages (games/\<id\>/index.html)

Each game also gets a standalone `play.html` generated by `deploy.sh` from `tools/web/play-template.html`. These work independently тАФ no app.html needed.

**Custom per-game pages**: Some games have hand-crafted `play.html` files instead of generated ones (e.g., `games/zork1-v4/play.html` has CSS atmospheric effects baked in). These are committed directly and are NOT overwritten by `deploy.sh`.

### Shared Parchment Library

All games share the same Parchment interpreter files from `lib/parchment/`. Standalone play pages reference them via `../../lib/parchment/`. The iframe-based `play.html` uses a relative `lib_path`. This means there's exactly one copy of the interpreter тАФ no per-game duplication.

Game binaries (`.ulx.js`) are the only per-game Parchment artifact. They are base64-encoded Glulx binaries wrapped in a JSONP callback: `processBase64Zcode('BASE64...')`.

## Game Registry (games.json)

Every game must have an entry in `games.json`. This file drives:
- The game selector in `app.html`
- The landing page cards in `index.html`
- Standalone play page generation in `deploy.sh`
- Sound control visibility (the `"sound": true` flag)

### Schema

```json
{
  "id": "mygame",
  "title": "My Game",
  "sourceLabel": "mygame.ni",
  "source": "games/mygame/story.ni",
  "binary": "games/mygame/mygame.ulx.js",
  "walkthrough": "games/mygame/walkthrough.html",
  "sound": true,
  "sourceBrowser": false,
  "versionLabel": "v1 тАФ Description",
  "card": {
    "title": "My Game",
    "meta": "A Subtitle",
    "description": "HTML description for the landing page card.",
    "versions": ["mygame-v0", "mygame-v1"]
  }
}
```

| Field | Required | Description |
|-------|----------|-------------|
| `id` | Yes | Unique identifier. Must match the `games/<id>/` directory name. |
| `title` | Yes | Display name in the game selector dropdown. |
| `sourceLabel` | Yes | Label for the source viewer tab. |
| `source` | Yes | Path to source file (relative to ifhub root). |
| `binary` | Yes | Path to base64-encoded game binary. |
| `walkthrough` | No | Path to walkthrough HTML page. Enables the walkthrough tab. |
| `sound` | No | Set to `true` to enable sound. Shows controls in app.html, injects scripts in standalone pages. |
| `sourceBrowser` | No | Set to `true` if source is an HTML page (loaded in iframe instead of fetched as text). |
| `versionLabel` | No | Shown in the game selector for versioned games (e.g., "v1 тАФ The Port"). |
| `card` | No | Landing page card. Games without a card don't appear on the landing page. |
| `card.versions` | No | Array of game IDs grouped under this card (for versioned games like Zork1). |

## Deployment (deploy.sh)

`deploy.sh` is a bash script that copies assets from each project's source directory into the `games/` directory, generates play pages and landing pages, and validates the results.

### What It Does

1. **Copies assets** тАФ `tools/deploy/copy-assets.py` reads `deploy` fields from `games.json` and copies source (`.ni`), binary (`.ulx.js`/`.gblorb.js`), and walkthrough files for each game
2. **Copies project-specific extras** тАФ ZIL source browser, Dracula BASIC source, Zork1 extra pages (bash, hardcoded)
3. **Generates play pages** тАФ `tools/deploy/generate-play-pages.py` reads `games.json` + `tools/web/play-template.html`, writes `games/<id>/play.html`
4. **Validates play pages** тАФ `tools/validate-web.sh` checks each generated page for common issues (missing files, bad format, unsubstituted tokens)
5. **Generates landing pages** тАФ `tools/deploy/generate-landing-pages.py` reads `games.json` + `landing-template.html`, writes `games/<base>/index.html`

### Configuration

All asset paths are defined in the `deploy` object of each `games.json` entry:

```json
{
  "id": "sample",
  "deploy": {
    "source": "projects/sample/story.ni",
    "binary": "projects/sample/web/lib/parchment/sample.ulx.js",
    "walkthroughDir": "projects/sample/web"
  }
}
```

Set `"source": null` for games without Inform 7 source (e.g., the ZIL-compiled v0).

### Running

```bash
cd ifhub/
bash deploy.sh
```

Output shows what was copied, with warnings for missing files and validation results.

## Sound System

Sound-enabled games use **native Glk/blorb sound** via Parchment 2025.1. Audio files (`.ogg`) are embedded in the game binary (`.gblorb`) and played by the interpreter via AudioContext and Glk sound channels. No external JavaScript overlay is needed.

For details on the blorb pipeline (`compile.sh --sound`, `generate-blurb.sh`, `setup-web.sh --blorb`), see `reference/sound.md`.

The previous JavaScript overlay system (MutationObserver-based text matching + Style_user1 triggers) has been archived to `reference/sound-overlay/`.

## Adding a New Game

### Prerequisites

The game must already be compiled and have a web player set up in its project directory. See `C:\code\ifhub\CLAUDE.md` for compiler paths and `tools/web/setup-web.sh` for web player setup.

### Steps

1. **Add to `games.json`** with game metadata and deploy paths:
   ```json
   {
     "id": "mygame",
     "title": "My Game",
     "sourceLabel": "mygame.ni",
     "source": "games/mygame/story.ni",
     "binary": "games/mygame/mygame.ulx.js",
     "deploy": {
       "source": "projects/mygame/story.ni",
       "binary": "projects/mygame/web/lib/parchment/mygame.ulx.js",
       "walkthroughDir": "projects/mygame/web"
     },
     "card": {
       "title": "My Game",
       "meta": "A Subtitle",
       "description": "Description for the landing page."
     }
   }
   ```

2. **Run deploy**: `bash deploy.sh`

3. **Commit and push** тАФ the `games/<id>/` directory and updated `games.json`

## Adding Sound to a Game

Use native Glk/blorb sound. See `reference/sound.md` for the full pipeline.

1. Add `.ogg` sound files to `projects/mygame/Sounds/`
2. Add sound declarations to `story.ni`: `Sound of X is the file "Y.ogg".`
3. Compile with `bash tools/compile.sh mygame --sound`
4. Set `"sound": "blorb"` in `games.json`
5. Run `bash deploy.sh`

## File Reference

### Hub-Level Files

| File | Purpose | When to Edit |
|------|---------|-------------|
| `index.html` | Landing page with game cards | Adding/removing games from the landing page |
| `app.html` | Main player UI with game selector, source viewer, sound controls | Changing the player UI, adding features |
| `play.html` | Parchment player loaded in iframe by app.html | Changing how games load, updating sound script loading |
| `landing-template.html` | Template for per-game landing pages | Changing landing page layout or styling |
| `games.json` | Game registry and deploy config | Adding/removing games, changing paths, enabling sound |
| `deploy.sh` | Asset deployment orchestrator | Adding project-specific extras (bash sections) |

### Per-Game Files (games/\<id\>/)

| File | Copied From | Purpose |
|------|-------------|---------|
| `play.html` | Generated from `tools/web/play-template.html` | Standalone play page |
| `story.ni` | Project's `story.ni` | Source code for source viewer |
| `*.ulx.js` / `*.z3.js` | Project's compiled binary | Game data for Parchment |
| `walkthrough.html` | Project's walkthrough dir | Walkthrough viewer page |
| `walkthrough.txt` | Project's walkthrough dir | Raw walkthrough commands |
| `walkthrough-guide.txt` | Project's walkthrough dir | Annotated walkthrough |
| `walkthrough_output.txt` | Project's walkthrough dir | Transcript of walkthrough run |

### Shared Parchment Library (lib/parchment/)

All 7 files are required. See `C:\code\ifhub\CLAUDE.md` for details on each file's role and what happens if it's missing.

### Source of Truth

| What | Authoritative Location | Copied To |
|------|----------------------|-----------|
| Parchment interpreter | `ifhub/lib/parchment/` | Referenced in-place (not copied) |
| Game source | `projects/<game>/story.ni` | `games/<id>/story.ni` |
| Game binary | `projects/<game>/web/lib/parchment/<game>.ulx.js` | `games/<id>/<game>.ulx.js` |
| Game metadata | `games.json` | Read by app.html, index.html, deploy.sh |
