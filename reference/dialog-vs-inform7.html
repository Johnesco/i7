1. Overview

Dialog Inform 7
Author Linus Åkesson Graham Nelson
First release 2018 2006
Current version 0.46 (beta) 10.1.2 (stable)
License BSD 2-clause (open source) Artistic License 2.0 (open source)
Paradigm Logic programming (Prolog-inspired) Natural language programming
Primary output Z-machine (z5/z8) or Å-machine Glulx (.ulx) or Z-machine (.z8)
Web playable Yes (via Å-machine bundle or Z-machine interpreters) Yes (via Parchment / Quixe)

Dialog is a rule-based language in the logic programming family, designed from the ground up for interactive fiction. It draws on Prolog's unification and backtracking model while providing a purpose-built standard library for world modeling and parsing.

Inform 7 is a natural-language programming system where source code reads like English prose. It compiles through Inform 6 (a C-like language) to produce virtual machine bytecode. Its Standard Rules provide an extensive, battle-tested world model.

2. Design Philosophy

Dialog: Lucidity, Minimalism, Performance

  • Lucidity — Story authors should be able to read, understand, and modify the standard library. The same language constructs handle everything from world modeling to parsing.
  • Minimalism — A few well-integrated concepts (predicates, rules, backtracking) provide expressive power without special-case syntax.
  • Performance — Designed from inception to generate compact, efficient code that can run on vintage hardware (including the Commodore 64).

Inform 7: Literacy, Expressiveness, Ecosystem

  • Natural language — Source code is meant to be readable by non-programmers. Rules are written as English sentences.
  • Rich world model — Decades of refinement in modeling rooms, containment, actions, scenes, and relations.
  • Extensibility — A large extension ecosystem and active community with thousands of published works.
Dialog prioritizes transparency and efficiency. Inform 7 prioritizes accessibility and literary expression. Both aim to make IF authoring approachable, but from opposite directions.

3. Syntax & Paradigm

The most immediately visible difference: Dialog uses a formal, symbolic syntax derived from logic programming, while Inform 7 uses natural English.

Hello World

Dialog
(program entry point)
  Hello, world!
Inform 7
"Hello" by "Author"

The Lab is a room.

When play begins:
  say "Hello, world!"

Dialog's entry point is a predicate. Inform 7 requires at minimum a title, an author, and a room (the world model demands a location for the player).

Rule Structure

Dialog
%% Rules are predicate definitions.
%% Multiple rules = multiple clauses,
%% tried in source order.

(descr #apple)
  The apple looks yummy.

(descr #door)
  The oaken door is sturdy.

(descr $)
  It looks unremarkable.
Inform 7
[ Rules are English sentences
  organized by rulebooks. ]

The description of the apple is
  "The apple looks yummy."

The description of the door is
  "The oaken door is sturdy."

Rule for examining something:
  say "It looks unremarkable."

Variables and Pattern Matching

Dialog
%% $ is wildcard, $Var captures.
%% # denotes objects.
%% * is "current topic" shorthand.

(colour #apple) red
(colour #sky) blue
(colour $) grey

(descr $Obj)
  It is (colour $Obj).
Inform 7
A thing has text called the colour.
The colour of the apple is "red".
The colour of the sky is "blue".
The colour of a thing is usually
  "grey".

Instead of examining something:
  say "It is [colour of the noun]."
Dialog's syntax is concise and unambiguous — every construct has one way to write it. Inform 7's syntax is more verbose but immediately readable as English, even to non-programmers. Dialog will feel natural to anyone with Prolog or functional programming experience; Inform 7 will feel natural to writers.

4. World Model

Defining Rooms and Things

Dialog
#kitchen
(name *)    kitchen
(room *)
(look *)    A warm room with
            copper pots on hooks.

#apple
(name *)    red apple
(edible *)
(* is #in #kitchen)
(descr *)   Crisp and inviting.
Inform 7
The Kitchen is a room. "A warm
room with copper pots on hooks."

The red apple is an edible thing
in the Kitchen. The description
is "Crisp and inviting."

In Dialog, objects are "thin" identifiers (#kitchen, #apple) — they have no inherent behavior. Everything about an object is expressed through predicate rules. The asterisk * is shorthand for the current topic object, reducing repetition.

In Inform 7, objects are declared with natural language assertions and automatically inherit behavior from their kind (thing, room, container, etc.).

Containment and Relations

Dialog
%% Three-part location syntax:
(#apple is #in #basket)
(#basket is #on #table)
(#table is #in #kitchen)

%% Move at runtime:
(now) (#apple is #in #pocket)

%% Relations: #in, #on, #under,
%% #behind, #heldby, #wornby,
%% #partof
Inform 7
The apple is in the basket.
The basket is on the table.
The table is in the Kitchen.

[ Move at runtime: ]
Now the apple is in the pocket.

[ Relations: in, on, part of,
  worn by, carried by,
  plus custom relations ]

Traits and Properties

Dialog
%% Traits are zero-argument predicates
(edible #apple)
(portable #apple)
(openable #box)
(container #box)

%% Custom traits:
(fragile #vase)
(material #vase) porcelain
Inform 7
The apple is edible and portable.
The box is an openable container.

[ Custom properties: ]
A thing can be fragile.
The vase is fragile.
A thing has text called material.
The material of the vase is
  "porcelain".
Both systems model equivalent world structures. Dialog is more explicit — every relationship is a predicate you write and can inspect. Inform 7 is more declarative at the surface level, with extensive implicit behavior inherited from kinds. Inform 7's custom relations (e.g., "X unlocks Y") go beyond what Dialog offers out of the box, though Dialog can model anything via predicates.

5. Actions & Rules

Action Processing Pipeline

Stage Dialog Inform 7
Accessibility check (refuse $Action) Accessibility rulebook
Automatic prerequisites (before $Action) Before rules
Override / redirect (instead of $Action) Instead rules
Block (prevent $Action) Check rules
Execute (perform $Action) Carry out rules
React (after $Action) After rules / Report rules

Custom Action Behavior

Dialog
%% Block eating the key:
(prevent [eat #key])
  That would be unwise.

%% Custom perform:
(perform [read #sign])
  The sign says "WELCOME".

%% Redirect:
(instead of [climb #stairs])
  (try [go #up])
Inform 7
Check eating the key:
  say "That would be unwise.";
  stop the action.

Carry out reading the sign:
  say "The sign says 'WELCOME'."

Instead of climbing the stairs:
  try going up.

Dialog represents actions as lists of words and objects: [give #apple to #eve]. This makes actions first-class data that can be inspected, stored, and transformed. Inform 7 actions are typed constructs with named parameters (noun, second noun).

Dialog's Group Actions

Dialog has a distinctive feature: group actions that allow combining multiple objects into a single action with custom narration.

(action [eat $] may group #ham with #cheese)

(group-perform [eat [#ham #cheese]])
  You savour the combination of ham and cheese.

Inform 7 doesn't have a direct equivalent — "taking all" processes items individually with separate rules.

6. Text Output & Formatting

Dialog
%% Paragraph and line breaks:
(par)
(line)

%% Styles:
(bold) Bold text (roman)
(italic) Italic text (roman)
(fixed pitch) Mono (roman)

%% Divs and spans with classes:
(div @quote) {
  To be or not to be.
}
(span @emphasis) { wow }

%% Style class definitions:
(style class @quote)
  font-style: italic;
  margin-top: 2em;

%% Case conversion:
(uppercase) (a $Obj)
Inform 7
[ Paragraph and line breaks: ]
say "[paragraph break]"
say "[line break]"

[ Styles: ]
say "[bold type]Bold[roman type]"
say "[italic type]Italic
  [roman type]"
say "[fixed letter spacing]Mono
  [variable letter spacing]"

[ No div/span system.
  Styling is via Glk hints
  set in I6 inclusions,
  or CSS in the web player. ]

[ Case conversion: ]
say "[The noun]"
say "[Noun in upper case]"

Dialog's div/span system with style classes gives authors more control over visual presentation directly in the source language, similar to writing semantic HTML. Inform 7 relies on the Glk layer for styling, which is more limited in the language itself but can be extensively customized via CSS in web players.

Hyperlinks and Resources

Dialog
%% Clickable text (appends to
%% input buffer):
Why don't you
  (link) {open the drawer}?

%% External resources:
(define resource @manual)
  feelies/manual.pdf
(link resource @manual) {Read it}

%% Embedded images:
(embed resource #lighthouse)
Inform 7
[ Hyperlinks require Glk
  extensions — not universally
  supported. Typically done
  via extensions or I6. ]

[ Images via figures: ]
Figure of Lighthouse is the file
  "lighthouse.png".
Display Figure of Lighthouse.

[ Sound via figures: ]
Sound of Thunder is the file
  "thunder.ogg".
Play Sound of Thunder.
Dialog has richer built-in support for hyperlinks, inline resources, and CSS-like styling. Inform 7 has better support for multimedia (images and sound in blorb packages) and benefits from Glk's standardized I/O layer, which is supported by more interpreters.

7. Parser & Understanding

Dialog
%% Dictionary words for objects:
#lamp
(name *) brass lamp
(dict *) lantern light

%% Grammar definitions use
%% predicate-based patterns:
(understand [switch on | $Obj])
  (perform [switch on $Obj])

%% Object resolution via:
(determine object $Obj)
  *($Obj is in scope)
(from words)
  *(dict $Obj)
(matching all of $Input)
Inform 7
The brass lamp is a device.
Understand "lantern" and "light"
  as the lamp.

Understand "switch on [something]"
  as switching on.

[ Object resolution is automatic
  based on scope, visibility,
  and disambiguation rules. ]

Understand "x [something]"
  as examining.
Does the player mean examining
  the lamp: it is likely.

Dialog's parser is implemented entirely in Dialog itself (it's part of the standard library, not a black box). Authors can read, modify, and extend every aspect of how input is parsed. Inform 7's parser is deeply integrated and handles disambiguation, pronoun tracking, and scope automatically, but the internals are harder to customize (they live in Inform 6 template code).

Dialog's parser is transparent and hackable — a major advantage for unconventional game designs. Inform 7's parser is more sophisticated out of the box (disambiguation, implicit actions, pronoun resolution) but harder to fundamentally alter.

8. NPCs & Conversation

Dialog
%% NPC movement and actions:
(perform [#npc going #north])
  Bob wanders north.

%% Ask/tell topics:
(perform [ask #bob about #weather])
  "Lovely day," Bob replies.

%% Choice-based conversation
%% is a built-in library feature.

%% Taking orders:
(perform [#bob, take #sword])
  Bob picks up the sword.
Inform 7
Every turn when Bob is in the
  Kitchen:
  if a random chance of 1 in 3
    succeeds:
    let D be a random adjacent
      room;
    try Bob going D.

Instead of asking Bob about
  "weather":
  say "'Lovely day,' Bob replies."

[ Choice-based conversation
  requires extensions like
  Hybrid Choices or
  Conversation Framework. ]

Persuasion rule for asking Bob
  to try taking:
  persuasion succeeds.
Dialog includes choice-based conversation as a built-in library feature. Inform 7 handles ask/tell natively but needs extensions for menu-based or choice-based dialogue trees. Both handle NPC autonomy (movement, independent actions), though Inform 7's "every turn" rules and scenes give more structure for complex NPC schedules.

9. Tooling & Deployment

Dialog Inform 7
IDE None (text editor + CLI) Full GUI IDE (Windows, macOS, Linux) with integrated source, index, skein, and testing
Compiler dialogc (C, open source) inform7 → inform6 (two-stage)
Debugger dgdebug — live source editing, hot reload, query inspection IDE skein, transcript comparison; no live debugger
Web bundler aambundle — generates a self-contained HTML/JS/CSS directory Parchment / Quixe — base64-encode the binary into a JS loader
Testing Debugger with @replay, @save, @restore Built-in Test me, RegTest, skein, walkthroughs
Extensions Include additional .dg files at compile time Formal extension system with versioning, the Public Library, and hundreds of community extensions

Dialog's debugger (dgdebug) is notably powerful: it runs source code directly without compilation, supports live editing with state preservation, and allows querying predicates interactively. This is closer to a REPL than anything Inform 7 offers.

Inform 7's IDE provides a richer overall authoring experience: the Index tab auto-generates maps, action tables, and cross-references; the Skein records play sessions as branching trees; and the built-in documentation is searchable with inline examples.

10. Performance & Targets

Dialog Inform 7
Primary VM Å-machine (custom) or Z-machine Glulx (32-bit) or Z-machine
Story size limit 256 KB (z5) / 512 KB (z8) / unlimited (Å-machine) Effectively unlimited (Glulx)
Vintage hardware Yes — runs on Commodore 64, 8-bit micros No — Glulx needs 32-bit; Z-machine output often exceeds vintage limits
Web interpreters Å-machine web runtime, Parchment (for Z-code output) Parchment (Quixe for Glulx, Bocfel for Z-machine)
Optimization Aggressive — global analysis, first-parameter dispatch, tail-call optimization Moderate — relies on I6 compiler optimizations
Multimedia Images (embeddable), hyperlinks; limited sound Images, sound (blorb), full Glk multimedia stack

Dialog's emphasis on performance is a defining characteristic. The compiler performs global analysis to choose optimal representations for each predicate, often converting what looks like a linear search through rules into a constant-time property lookup. This makes Dialog uniquely suited for resource-constrained platforms.

Inform 7's Glulx target has no practical size limits and supports a richer multimedia stack (images, multi-channel sound via blorb), making it better suited for large, media-rich works.

11. Ecosystem & Community

Dialog Inform 7
Published games Handful (including award-winning The Impossible Bottle) Thousands (dominant parser-IF language since 2006)
Extensions Minimal — standard library covers most needs Hundreds in the Public Library (conversation, UI, puzzles, etc.)
Documentation Comprehensive manual (Åkesson) Extensive — Writing with Inform, The Recipe Book, community wikis
Community Small, growing (intfiction.org) Large, established (intfiction.org, Discord, Reddit, IFTF)
Competition wins The Impossible Bottle (IFComp 2020, 1st place) Numerous IFComp and XYZZY winners across two decades
Learning resources Official manual, example code Books, tutorials, university courses, YouTube, example projects
Inform 7 has an overwhelming ecosystem advantage. Dialog is a younger language with a smaller but dedicated community. The Impossible Bottle (2020 IFComp winner) demonstrated that Dialog can produce world-class IF, but the learning resources and community support for Inform 7 are much more extensive.

13. Which Should You Choose?

Choose Dialog if you…

  • Have programming experience (especially logic programming or Prolog)
  • Want to understand and modify every layer of the system, including the parser
  • Are targeting vintage hardware or need extremely compact output
  • Prefer a powerful debugger with live-editing capabilities
  • Want built-in choice-based conversation without extensions
  • Value a minimal, coherent language over a large ecosystem

Choose Inform 7 if you…

  • Are new to programming or prefer writing in natural English
  • Want a full IDE with visual tools (map, index, skein)
  • Need multimedia support (embedded sound, images, blorb packaging)
  • Are building a large, complex game that may exceed Z-machine limits
  • Want access to a large extension library and active community
  • Need a stable, mature platform for long-term projects

Consider both if you…

  • Want to prototype in Dialog (fast compile, great debugger) and publish in either
  • Are writing a parser game with unconventional mechanics — Dialog's transparent parser may be easier to bend to unusual designs
  • Are interested in the craft of IF system design — studying both languages will deepen your understanding
Neither language is universally "better." Dialog is a sharper, more transparent tool for authors who think like programmers. Inform 7 is a more welcoming, more feature-rich environment for authors who think like writers. The best IF works have been created in both.
